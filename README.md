# Forex Signal Analyzer

A production-ready Progressive Web App (PWA) for rule-based technical analysis on **EUR/USD** and **GBP/JPY** forex pairs.

> **Educational tool only.** Not financial advice. All signals are based on deterministic, auditable technical rules.

---

## Features

| Feature | Details |
|---------|---------|
| **Pairs** | EUR/USD · GBP/JPY |
| **Timeframes** | 5m · 15m · 1h · 4h · 1D |
| **Chart** | TradingView Lightweight Charts with EMA20/50 overlays |
| **Indicators** | EMA, RSI, MACD, ATR, Support/Resistance, Candlestick Patterns |
| **Signal Engine** | Weighted scoring → STRONG BUY / BUY / HOLD / SELL / STRONG SELL |
| **Trade Setup** | ATR-based entry, stop-loss, take-profit, risk/reward |
| **Backtesting** | Walk-forward, equity curve, win rate, profit factor |
| **Signal History** | Local persistence, CSV export |
| **Settings** | All indicator thresholds configurable and persisted |
| **Alerts** | In-app + optional browser (OS) notifications |
| **PWA** | Installable on Android, iOS, desktop Chrome |
| **Offline** | Service worker caches app shell + recent data |

## Signal Engine Logic

The signal engine computes a **weighted score** across five dimensions:

| Dimension | Max | Logic |
|-----------|-----|-------|
| **Trend** | +-2 | EMA20 vs EMA50 gap and direction |
| **Momentum** | +-2 | RSI zone + MACD cross/histogram direction |
| **Breakout** | +-2 | S/R proximity, breakouts, breakdowns |
| **Pattern** | +-1 | Bullish/bearish candlestick patterns |
| **Volatility** | 0/-2 | ATR spike penalty |

**Score to Signal mapping:**
```
>= +4   =>  STRONG BUY
>= +2   =>  BUY
-2..+2  =>  HOLD
<= -2   =>  SELL
<= -4   =>  STRONG SELL
```

If confidence falls below `minConfidence` (default 55%), the signal is downgraded to HOLD.

### Example BUY Signal

> "EMA20 strongly above EMA50 — bullish trend. RSI is 58.3 — bullish momentum. MACD above signal — bullish. Price near support zone — potential bounce."

### Example SELL Signal

> "EMA20 below EMA50 — downtrend active. RSI is 38.1 — weak/bearish momentum. MACD below signal — bearish. Price broke through support — bearish breakdown."

## Trade Setup

Stop loss and take profit are calculated using ATR:

```
Stop Loss   = Entry +/- ATR x 1.5
Take Profit = Entry +/- ATR x 2.5
Risk/Reward = 1:1.67 minimum (configurable)
```

---

## Backtesting

The backtester uses a **walk-forward** approach:
- Only uses data available at candle time (no lookahead bias)
- Re-evaluates signal every 4 candles
- Manages simulated trades with SL/TP hit detection
- Reports: total trades, win rate, profit factor, total R, max drawdown, equity curve

## Plugging in Real Data

Replace the sample data provider by implementing the `DataProvider` interface:

```typescript
// lib/data/myRealProvider.ts
export const myRealProvider: DataProvider = {
  getName: () => "My Broker API",
  async getCandles(pair, timeframe) {
    const res = await fetch(`https://api.example.com/candles?pair=${pair}&tf=${timeframe}`);
    return res.json();
  },
};
```

Then activate it:
```typescript
import { setDataProvider } from "@/lib/data/provider";
setDataProvider(myRealProvider);
```

## Settings

Open the **Settings** tab in the app to customize:
- RSI oversold/overbought levels
- EMA fast/slow periods
- ATR multipliers for SL and TP
- Minimum confidence threshold
- Volatility filter sensitivity
- Browser notification toggle

All settings persist via `localStorage`.

## Extending the App

- **Real data API**: Replace `sampleDataProvider` with any forex REST API
- **More indicators**: Add files in `lib/indicators/` and wire into `signalEngine.ts`
- **More pairs**: Add to `FOREX_PAIRS` in `types/index.ts`
- **IndexedDB**: Replace `lib/storage/storage.ts` with `idb` (already installed)
- **Telegram alerts**: Extend `lib/pwa/notifications.ts`
- **Multi-TF confirmation**: `higherTFConfirms()` already implemented in `signalEngine.ts`
